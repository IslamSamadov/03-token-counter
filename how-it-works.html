How an LLM Works

  1. 1

    User Input

    The user types a prompt into the chat interface and submits it. The raw text is sent to the model's API as a plain-text string, ready to be processed.

  2. 2

    Tokenisation

    The raw text is broken into tokens — small chunks of roughly 3–4 characters. For example, "Hello, world!" becomes ["Hello", ",", " world", "!"]. Each unique token is mapped to a specific integer called a token ID.

  3. 3

    Embedding

    Each token ID is looked up in an embedding table and converted into a high-dimensional vector — a list of numbers that encodes the semantic meaning of the token in a form the neural network can process.

  4. 4

    Attention Layers

    The embedded tokens pass through dozens of transformer layers. Each layer uses self-attention to weigh how much every token should influence every other token, allowing the model to capture context and relationships across the entire prompt.

  5. 5

    Prediction

    After processing through all layers, the model produces a probability distribution over its entire vocabulary — a score for every possible next token. The token with the highest probability (or one sampled from the distribution) is selected as the next output token.

  6. 6

    Output Generation

    The predicted token is appended to the sequence and the process repeats — tokenise, embed, attend, predict — one token at a time until a stop condition is met. The final string of tokens is decoded back into human-readable text and streamed to the user.
